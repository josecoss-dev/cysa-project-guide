Lab environment

Setting up your virtual machines

No spare hardware needed. All 8 projects run on VMs inside your existing laptop or desktop. Here's exactly how to set it up from zero.

📦
Step 1 — Install VirtualBox
Free, cross-platform hypervisor. Runs on Windows, Mac, and Linux. Lets you run multiple OSes simultaneously on your existing machine.
  • Go to virtualbox.org/wiki/Downloads
  • Download for your host OS
  • Also download the Extension Pack (same page)
  • Install VirtualBox first, then the Extension Pack
↗ virtualbox.org/wiki/Downloads

⚡
Low-RAM tips (under 16 GB)
You don't need all VMs running at once. Run one project at a time and use snapshots to save state.
  • Take a snapshot after each working state — revert if things break
  • Only run the VMs needed for the current project
  • Projects 3, 5, 7 run entirely on your host — no extra VMs needed
  • Project 4 (Splunk) can run on your host machine directly
  • Reduce Wazuh server RAM to 4 GB if host has only 12 GB total
  • Use VirtualBox's "Save State" instead of shutting down — faster resume
Clone any VM before starting a new project phase. Clones are cheap and save you from rebuilding from scratch.
🌐
Network configuration
All lab VMs should talk only to each other, never to your real network or the internet unless absolutely needed.
  • VirtualBox → File → Tools → Network Manager
  • Create Host-Only network: 192.168.56.0/24
  • Set all lab VMs to this adapter
  • Wazuh server: 192.168.56.10 (set static)
  • Linux agent: 192.168.56.11
  • Windows agent: 192.168.56.12
  • Metasploitable: 192.168.56.20
Add a second NAT adapter to your Wazuh server only, for package installation. Remove it after setup is complete.
💾
Storage planning
Plan your disk usage before creating VMs. Dynamic allocation grows as needed but reserve space upfront.
Wazuh server60 GB
Linux agent30 GB
Windows agent60 GB
Metasploitable 28 GB (fixed)
Snapshots (each)~5–10 GB
Total recommended200 GB free
Store VMs on an SSD if possible — HDD VMs are noticeably sluggish, especially Wazuh's OpenSearch indexer.
